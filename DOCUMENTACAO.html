🏠 1. Visão Geral do Sistema

O Sistema de Produção é uma aplicação web usada para registrar, acompanhar e analisar a produção de equipes de campo da D.B. Machado LTDA. Ele substitui planilhas manuais por um sistema centralizado com controle de acesso.

O que o sistema faz
✅ Lançamento diário de produção
✅ Relatórios mensais e por equipe
✅ Exportação de dados
✅ Planejamento de campo (RMB/RIP)
✅ Controle de usuários e acessos
✅ Cadastro de contratos, equipes e colaboradores
Quem usa
👑 Super Admin — acesso total
👤 Usuário comum — lançamento e relatórios
🗺️ Planejamento — somente página de planejamento

Cada usuário pode ter acesso a contratos específicos.

⚙️ 2. Tecnologias Usadas

Você não precisa ser especialista em tudo, mas entender o papel de cada tecnologia ajuda a saber onde mexer.

⚛️ React — A interface do usuário
React é uma biblioteca JavaScript que constrói as telas do sistema. Em vez de escrever HTML puro, você escreve componentes — como peças de LEGO que se encaixam para formar a página.
Exemplo simples:
// Um componente React é uma função que retorna HTML
function BotaoSalvar() {
  return (
    <button onClick={() => alert('Salvou!')}>
      Salvar
    </button>
  )
}
🗄️ Supabase — O banco de dados na nuvem
Supabase é onde os dados ficam guardados. Ele fornece o banco de dados PostgreSQL (planilhas relacionais) e também cuida do login dos usuários. O sistema se comunica com o Supabase para salvar e buscar dados.
Como o sistema busca dados:
// Busca todos os registros de produção
const { data } = await supabase
  .from('f_prod_registro')
  .select('*')
  .order('data', { ascending: false })
⚡ Vite — A ferramenta de construção
Vite é quem "compila" o código React em arquivos que o navegador consegue entender. Também serve o sistema localmente durante o desenvolvimento. Para rodar o projeto, você usa o comando npm run dev.
🎨 CSS — O visual do sistema
O arquivo index.css contém todos os estilos visuais: cores, tamanhos, espaçamentos. Classes como card, btn, pagina são usadas nos componentes React.

Produção

Planejamento

🗺️ Planejamento (RMB + RIP)
Página com duas abas que carregam arquivos HTML independentes dentro de um iframe. Cada arquivo HTML tem seu próprio sistema de mapa (Leaflet), quadro semanal e exportação. Os dados vêm direto do Google Sheets via API pública.

Por que iframe? Os arquivos HTML são sistemas completos por si só (com mapa, JavaScript próprio). O iframe é a forma mais simples de integrá-los ao sistema React sem reescrever tudo.

🧩 5. Componentes Reutilizáveis

Componentes são peças de interface usadas em várias páginas. Quando você altera um componente, a mudança aparece em todos os lugares que o usam.

Sidebar.jsx — Menu lateral
O menu lateral com todos os links de navegação. Para adicionar um item no menu, edite o array itensMenu no início do arquivo. O menu filtra automaticamente itens baseado no perfil do usuário (ex: perfil "Planejamento" vê só o item de planejamento). 
// Para adicionar um novo item no menu, adicione aqui:
const itensMenu = [
  { secao: 'MENU' },
  { caminho: '/', icone: '📊', label: 'Dashboard' },
  // ... outros itens
]
TabelaCRUD.jsx — Tabela com ações
Tabela genérica usada nas páginas de configuração (Contratos, Equipes, etc.) com botões de Editar e Excluir embutidos. Também exporta o componente Modal usado nos formulários de criação.
useAuth.jsx — Hook de autenticação
Não é uma tela, mas um "hook" — uma função especial do React que qualquer componente pode usar para saber quem está logado. Fornece:
• usuario — dados do Supabase Auth (email, id)
• perfil — dados da tabela d_auth_user (nome, foto, perfil de acesso)
• carregando — true enquanto verifica o login
• sair() — função para fazer logout

🗄️ 6. Banco de Dados

O banco de dados fica no Supabase. Cada tabela é como uma planilha. Veja as principais:

Tabela Para que serve Campos principais
f_prod_registro Registro de produção (cada lançamento) data, contrato_id, equipe_id, criado_por_id
f_prod_atividades Atividades dentro de cada registro registro_id, atividade_id, quantidade, valor_total
f_prod_colaboradores Colaboradores em cada registro registro_id, equipe_id, colaborador_id
d_contratos Contratos da empresa descricao, ativo, logica_contrato
d_equipes Equipes de campo sistema_producao, contrato_id, tipo_equipe_id
d_colaboradores Colaboradores (pessoas) nome, equipe_id
d_atividades Tipos de atividade descricao, unidade
d_auth_user Perfil dos usuários do sistema uuid, nome, email, role_id, foto_url
d_auth_roles Perfis de acesso (Super Admin, Planejamento...) name
d_auth_contratos Quais contratos cada usuário acessa user_uuid, contrato_id
config_campos Definição dos campos dinâmicos do formulário label, tipo, opcoes
config_campos_contrato Quais campos aparecem em cada contrato/equipe campo_id, contrato_id, tipo_equipe_id, ordem
O prefixo f_ nas tabelas significa "fato" (dados transacionais). O prefixo d_ significa "dimensão" (dados cadastrais). Essa é uma convenção de banco de dados chamada esquema estrela.

Funções RPC no Supabase

Algumas operações precisam de permissões especiais (como criar usuários no sistema de autenticação). Elas são feitas via funções armazenadas no banco:

FunçãoO que faz
criar_usuario_auth(email, senha, nome, role_id)Cria um usuário no Auth e na tabela d_auth_user de forma atômica
deletar_usuario_auth(uuid)Remove usuário do Auth, d_auth_user e d_auth_contratos
alterar_senha_usuario(uuid, senha)Altera a senha de um usuário

🔐 7. Autenticação e Perfis de Acesso

O sistema usa o Supabase Auth para gerenciar login. O fluxo é:

  1. Usuário entra com e-mail e senha
    O Supabase verifica as credenciais na tabela auth.users.
  2. Sistema carrega o perfil
    O hook useAuth busca os dados do usuário na tabela d_auth_user e seu perfil de acesso em d_auth_roles.
  3. Rotas são protegidas por perfil
    O componente RotaProtegida verifica se o usuário está logado. O componente RotaSuperAdmin verifica se é Super Admin.
  4. Dados são filtrados por contrato
    Cada usuário só vê dados dos contratos que tem acesso (tabela d_auth_contratos).

🔧 8. Como Fazer Alterações Comuns

Adicionar um item no menu lateral

Abra src/components/Sidebar.jsx e edite o array itensMenu:

// Adicione um novo item assim:
{ caminho: '/minha-pagina', icone: '⭐', label: 'Minha Página' }

Alterar uma cor ou estilo

Abra src/index.css. As classes principais são:

Classe CSSOnde é usada
.cardCaixas brancas com sombra
.btn-primarioBotão azul principal
.btn-secundarioBotão cinza secundário
.paginaÁrea de conteúdo de cada página
.campo-inputCampos de texto
.campo-selectDropdowns

Adicionar uma nova rota (página)

  1. Crie o arquivo da página
    Ex: src/pages/MinhaPagina.jsx
  2. Importe no App.jsx
    import MinhaPagina from './pages/MinhaPagina'
  3. Adicione a rota no App.jsx
    <Route path="/minha-pagina" element={<RotaProtegida><MinhaPagina /></RotaProtegida>} />
  4. Adicione o link no sidebar
    Edite itensMenu no Sidebar.jsx

Alterar um texto ou rótulo

Procure o arquivo .jsx da página correspondente e edite o texto diretamente. Use Ctrl+F para buscar o texto no arquivo.

Adicionar uma nova coluna em uma tabela do banco

  1. No Supabase
    Vá em Database → Tables → selecione a tabela → Add Column
  2. No código React
    Adicione o campo no .select() da query e exiba na tela
Sempre teste as alterações em ambiente local antes de publicar. Use npm run dev para rodar o sistema localmente e acesse http://localhost:5173.

📚 9. Conceitos de Programação — Aprendendo com o Sistema

Conceito 1: Estado (State)

Estado é uma variável que, quando muda, atualiza automaticamente a tela. No React, usamos o hook useState:

// Declara uma variável de estado
const [carregando, setCarregando] = useState(false)

// Para mudar o valor e atualizar a tela:
setCarregando(true)   // mostra spinner
setCarregando(false)  // esconde spinner
No sistema, quase toda página tem estados como carregando, erro, dados. Esses três controlam o que o usuário vê.

Conceito 2: Efeito (useEffect)

O useEffect executa código quando a página carrega ou quando algo muda. É usado para buscar dados do banco:

// Executa UMA VEZ quando a página abre
useEffect(() => {
  buscarDados()
}, [])  ← o [] vazio significa "só uma vez"

// Executa toda vez que 'contratoId' muda
useEffect(() => {
  buscarDados()
}, [contratoId])

Conceito 3: Async/Await — Código assíncrono

Buscar dados do banco leva tempo. O async/await espera a resposta antes de continuar:

async function buscarContratos() {
  setCarregando(true)

  const { data, error } = await supabase
    .from('d_contratos')
    .select('*')

  if (error) {
    setErro(error.message)  // mostra o erro
  } else {
    setContratos(data)      // salva os dados
  }

  setCarregando(false)
}

Conceito 4: Componente — Reutilização de código

Componentes são funções que retornam HTML. Você pode passá-los parâmetros (props) para personalizar:

// Definindo o componente
function CartaoUsuario({ nome, email }) {
  return (
    <div className="card">
      <div>{nome}</div>
      <div>{email}</div>
    </div>
  )
}

// Usando o componente
<CartaoUsuario nome="João" email="joao@empresa.com" />

Conceito 5: Map — Renderizar listas

Para mostrar uma lista de itens, usamos .map() para transformar cada item em HTML:

// Renderiza um item da lista para cada contrato
{contratos.map(contrato => (
  <div key={contrato.id} className="card">
    {contrato.descricao}
  </div>
))}
O key={contrato.id} é obrigatório no React para identificar cada item da lista. Sempre use um identificador único.

O arquivo supabaseClient.js

Este arquivo cria a conexão com o Supabase. Todos os outros arquivos importam ele para acessar o banco:

// supabaseClient.js — a "ponte" para o banco de dados
import { createClient } from '@supabase/supabase-js'

export const supabase = createClient(
  'https://sua-url.supabase.co',
  'sua-chave-publica'
)
Nunca coloque a chave service_role do Supabase no código frontend. Use sempre a chave anon (pública). A service_role tem acesso total ao banco sem restrições.

Como o Git/GitHub é usado

Cada alteração no código é salva com um commit no Git e enviada ao GitHub. Isso permite:
